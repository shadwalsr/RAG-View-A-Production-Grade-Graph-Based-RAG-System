## 2. Entity Resolution & Deduplication (`src/resolver.py`)

A fundamental flaw of LLM-based extraction is the creation of duplicate nodes (e.g., extracting "WhySchool", "WhySchool Academy", and "WhySchool Startup" as distinct entities). We solved this graph fragmentation.

- **Semantic Clustering**: The resolver uses Cypher's `vector.similarity.cosine` against our Gemini embeddings to find node pairs with a similarity score `>= 0.92`. A custom Python connected-components algorithm groups overlapping matches into definitive clusters.
- **Canonical Merging**: A complex transaction selects the node with the most `source_chunk_ids` as the primary entity. It appends the names of duplicates as `aliases`, combines provenance arrays to ensure no text links are lost, and uses `apoc.refactor.mergeNodes` (with a pure Cypher fallback) to seamlessly rewire all incoming and outgoing graph relationships.
- **Automation**: This process runs automatically at the end of the ingestion pipeline.

## 3. The Retrieval Router (`src/retriever.py`)

This is the system that answers user queries by intelligently querying both flat text and structured graph stores.

- **Query Embedding**: The user's question is instantly converted into a 768-dimensional vector via Gemini.
- **Hybrid Data Retrieval**: The system hits the `HybridStore` (ChromaDB + BM25) to retrieve the top text excerpts and parses their metadata to extract linked `entity_ids`.
- **Graph Expansion**: It queries Neo4j for the immediate 1-hop neighborhood of those linked entities. Simultaneously, it performs a pure semantic vector search directly against the Neo4j graph using the query's embedding to find related entities the text search might have missed.
- **Context Fusion**: The text excerpts and graph relationships are concatenated into a highly dense context prompt.

## 4. Q/A Synthesis (`src/qa.py`)

- A lightweight wrapper around `gemini-2.0-flash` was built. It accepts the massive fused context block generated by the Retrieval Router and strictly synthesizes an answer using *only* the provided information, effectively mitigating hallucinations.

## 5. Pipeline Hardening & Utilities

- **Document Ingestion (`ingest.py`)**: Built a drop-folder ingestion utility (`data/raw/`) that automatically pipes `.txt` and `.pdf` files through the entire text chunking, graph extraction, and vector embedding pipeline. Integrated `pypdf` securely within the virtual environment.
- **Proactive Rate Limiting**: Implemented proactive `time.sleep(5)` pacing mechanisms within the pipeline loop. This spaces out API calls to comfortably sit beneath the Gemini Free Tier's strict 15 Request Per Minute (RPM) ceiling, avoiding catastrophic `429 RESOURCE_EXHAUSTED` errors during multi-chunk operations.
- **Environment Tooling**: Configured `.vscode/settings.json` to automatically inject `.env` variables into integrated terminals.
